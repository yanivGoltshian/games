# הרפתקת למידה

אפליקציית PWA מקומית, חמה וידידותית למגע לילדים צעירים שעדיין אינם קוראים. הפרויקט בנוי ב־Vite + React + TypeScript, מותאם קודם כול לאייפד ולטלפון, וכולל שבעה משחקים פעילים עם התקדמות אדפטיבית מקומית בלבד. באזור המבוגרים אפשר להגדיר שם אישי ולהוסיף תמונות משפחתיות לפאזלים; הכול נשמר רק במכשיר.

## פקודות

```bash
npm install
npm run dev
npm run test
npm run lint
npm run typecheck
npm run build
npm run preview
```

## התקנה על אייפד (הערה למבוגרים בלבד)

### Safari
1. פותחים את האפליקציה.
2. לוחצים על כפתור השיתוף.
3. בוחרים **הוספה למסך הבית**.
4. מאשרים את השם “הרפתקת למידה”.

### Chrome באייפד
1. פותחים את התפריט או Share.
2. בוחרים **Add to Home Screen**.
3. מאשרים את ההתקנה.

אחרי טעינה ראשונה, ה־service worker שומר קבצים מקומיים כך שאפשר לשחק גם בלי חיבור רשת. כל ההערות האלה על התקנה, אופליין וקול מופיעות רק במסך המבוגרים באפליקציה עצמה (אזור למבוגרים), ולעולם לא במסך הילד.

## מה יש בפנים

- **בית לילד**: שבעת הפורטלים הפעילים גלויים יחד במיקומים יציבים. אין משחקים נעולים ואין צורך לגלות מחוות החלקה. כל פורטל הוא כפתור מגע גדול עם סצנה ויזואלית ייחודית והשקה בלחיצה אחת.
- **7 משחקים פעילים**, עם רמה 1 מוגבלת בכוונה למספר קטן של פעולות וברירות:
  - שומעים ובוחרים
  - כמה יש?
  - ממיינים צבעים וצורות
  - פאזל קטן עם snap סלחני
  - זוגות זיכרון
  - זוגות מספרים
  - החייזר המבולבל, שפועל ללא מגע אחרי ההפעלה
- **החלטת מוצר לשפה**: מדף התקשורת ופעילויות השפה הקודמות הוסרו בכוונה כי הן בסיסיות מדי לגיל היעד ואינן מספקות לולאת משחק ואוצר מילים עשירים מספיק. הבית נשאר כרגע עם שבעה משחקים עד לבחירת משחק שפה חדש ומתאים יותר.
- **פאזלים מתמונות משפחתיות מקומיות**: מבוגר יכול לבחור תמונות דרך בורר התמונות או הקבצים באייפד. לפני פענוח הפיקסלים נבדקות מידות התמונה מתוך מבנה הקובץ, ותמונה גדולה מכדי לעבד בבטחה נדחית. הדפדפן מפענח את התמונה ישירות לגודל מוקטן כשהמכשיר תומך בכך, מצייר מחדש את הפיקסלים בכיוון הנכון, מגביל את הצד הארוך ל־1600 פיקסלים ושומר עותק JPEG חדש ב־IndexedDB. קובץ המקור, שם הקובץ, EXIF, מיקום ופרטי מקור אחרים אינם נשמרים. הספרייה מוגבלת ל־24 תמונות, זמינה אופליין רק באותו מכשיר ומופיעה רק באזור המבוגרים ובתוך משחק הפאזל.
- **מסך משחק לילד**: משימה אחת ברורה בכל זמן נתון, ובראש המסך רק שני כפתורים גדולים – חזרה לבית והשמעה חוזרת. שער המבוגרים אינו מופיע כלל בתוך משחק.
- **שפה חזותית מקומית**: 21 תמונות מושגים מציאותיות שנוצרו במיוחד עבור האפליקציה נשמרות מקומית כ־WebP ומופיעות לצד ספריית SVG מקורית, עשירה וממדית. 14 המושגים החדשים זמינים בהאזנה, בזיכרון, בספירה ובפאזלים מבוססי תמונה בהתאם לרמה. הפורטלים, סצנות הפאזל המקוריות, המסקוט, הקונפטי והבקרות הם מקוריים; אין אימוג׳י ואין hotlink.
- **חגיגת הצלחה עדינה**: כל תשובה נכונה מציגה חיזוק לא־חוסם עם המסקוט, קונפטי, צליל וריטוט אופציונלי. המעבר לסיבוב הבא מתרחש רק אחרי שהיעד והשבח סיימו להישמע, או בלחיצה מפורשת להתקדמות. לאחר רצף הצלחות קטן מופיעה חגיגה מעט עשירה יותר – בלי לחץ ובלי מונים.
- **שבחים מרכזיים ודטרמיניסטיים** ב־`src/content/praise.ts`: מזהים יציבים, בחירה טהורה ונבדקת (`praise.test.ts`), עברית ואנגלית תואמות לשפת הלמידה הפעילה, ורצף דיבור שתמיד משמיע קודם את שם היעד/התשובה ורק אז את השבח.
- **עידוד לימודי אחרי ניסיון נוסף**: אין צליל כישלון, סימון אדום או שפה של טעות. כל משחק מציג רמז חזותי עדין, משמיע קודם מודל קצר ורלוונטי ורק אחריו משפט עידוד מתחלף. במשחק הספירה סופרים יחד ומדגישים כל עצם בתורו; בזיכרון אי־התאמה מוצגת כחלק טבעי מהחיפוש.
- **התקדמות אדפטיבית** ב־localStorage: ניסיונות, הצלחות, רמת שליטה, עלייה עדינה ברמה וחזרה על מושגים חלשים – מוצגת רק במסך המבוגרים.
- **אזור למבוגרים** שנפתח בלחיצה ארוכה של כ־2.7 שניות על גלגל השיניים, ורק שם.
- קול דרך SpeechSynthesis, צלילי חיזוק דרך Web Audio, ריטוט קליל אופציונלי, מצב שקט, פחות תנועה (גם לפי הגדרת מבוגר וגם לפי `prefers-reduced-motion`), והגדרות שפה.
- PWA מקומי בלבד, בלי אנליטיקות, בלי חשבונות ובלי קישורים חיצוניים.

## ארכיטקטורה קצרה

- `src/art` ו־`public/assets/vocabulary` – ספריית אמנות מקורית לממשק ולחגיגות לצד אוצר מילים פוטוריאליסטי מקומי שנוצר במיוחד עבור האפליקציה. כל מושג לימודי וסצנת פאזל משתמשים בתמונה מציאותית עם מזהי `data-content-id` ו־`data-art-id` יציבים.
- `src/content` – מטא־דאטה מרכזי למושגים, משחקים, סצנות פאזל, בנק שבחים ובנק עידוד מקומי עם מזהים יציבים.
- `src/domain` – RNG דטרמיניסטי, לוגיקת סיבובים (כולל אילוצי רמה 1), דקדוק כמויות, מדרגות רמז, התקדמות, ולידציה/מיגרציה של persistence.
- `src/services` – שמירת התקדמות ב־localStorage, ספריית תמונות פרטית וממוסגרת ב־IndexedDB, המרת תמונות מקומית דרך פיקסלים, תור הקראה סדרתי עם מזהי בקשות/עדיפויות, איחוד משוב כפול ורצפי "מודל ואז עידוד" ו"יעד ואז שבח" שממתינים לסיום אמיתי, וצלילי חיזוק/ריטוט.
- `src/components` – מעטפת UI לילד (בית, מסגרת משחק, חגיגת הצלחה), שער למבוגרים, פאנל מבוגרים, עזרי גרירה משותפים.
- `src/games` – משחקי הילד.
- `public/sw.js` ו־`public/manifest.webmanifest` – יכולות PWA ואופליין.

## פרטיות

כל המידע נשמר רק על המכשיר המקומי. אין שרת, אין שליחת נתונים, אין איסוף אנליטיקות, אין חשבונות ואין אחסון חיצוני. תמונות מיובאות אינן מועלות, מסונכרנות, נרשמות ללוגים או נכנסות למטמון ה־service worker. רק עותק JPEG שנבנה מחדש מפיקסלים וללא מטא־דאטה נשמר ב־IndexedDB המקומי; מחיקתו באזור המבוגרים מסירה אותו מן האפליקציה.

### שיחה ומשחק משותף בעולם האמיתי

שום אפליקציה, כולל זו, אינה תחליף לשיחה, הקראה, ומשחק משותף עם מבוגר. הסיבובים במשחקים קצרים בכוונה, ואחרי רצף הצלחות קטן מופיעה חגיגה עם רגע חזותי רגוע שמזמין לצאת לשחק יחד. מומלץ שההורה/המטפל יהיה נוכח, ישתתף וימשיך את החוויה בעולם האמיתי – שיחה, הצבעה על עצמים אמיתיים, חזרה על מילים וקריאה משותפת תורמות הרבה יותר מזמן מסך עצמאי.

## מנגנון הקריינות

בדפדפנים רגילים ההקראה משתמשת ב־SpeechSynthesis של הדפדפן (Safari/Chrome), ולכן:
- איכות הקול תלויה בקולות המותקנים במכשיר.
- ייתכן שבמכשירים שונים יהיו הבדלים בהגייה ובזמינות הקולות.
- אם אין קול זמין, המשחקים עדיין עובדים במלואם עם תמונות ומגע – שום דבר לא תלוי בקול בלבד.

השירות משמיע בקצב לימודי מתון (`0.72` בעברית, `0.76` באנגלית), בוחר באופן דטרמיניסטי קול מקומי תואם־שפה כשקיים, ומסדר את כל ההקראות בתור יחיד. יעד פעיל אינו נקטע בגלל render או טיימר; רק חזרה יזומה או ניווט מפורש מפסיקים אותו. זה מונע חיתוך תוכנתי, אך אינו מבטיח איכות הגייה זהה בין גרסאות iPadOS או בין קולות שונים.

במצב PWA מותקן באייפד או באייפון השירות אינו תלוי ב־SpeechSynthesis. הוא פותח מתוך מחוות המשתמש הראשונה את אותו `AudioContext` שמפיק את צלילי המשחק, ומשמיע ממנו מקטעים מוקלטים מקומיים. כל 1,470 המקטעים בשלוש גרסאות הקול ארוזים בשלושה קובצי שמע ובמפת מקטעים תחת `public/speech/`. קבצים אלה נכללים מראש במטמון של ה־service worker, ולכן הקריינות פועלת גם ללא חיבור לרשת.

פתיחת מנגנון השמע רק מחזירה לפעולה את ה־`AudioContext`. קובץ הקול של השפה המבוקשת נטען ומפוענח רק בהקראה הראשונה, ובכל זמן נשמר בזיכרון לכל היותר קובץ שפה מפוענח אחד. מעבר שפה משחרר את ההפניה לקובץ הקודם, ובקשות מקבילות לאותה שפה חולקות פעולת טעינה ופענוח אחת.

`src/content/recordedSpeechCatalog.ts` מפיק את רשימת המשפטים מאותם מקורות ובוני משפטים שבהם משתמשים המשחקים. בדיקת כיסוי מחייבת התאמה מלאה בין הקטלוג למפה שנוצרה. לאחר שינוי תוכן דיבור מייצרים מחדש את הנכסים המקומיים:

המחולל משתמש ב־Azure Neural Text to Speech דרך Microsoft Entra ID בלבד. נדרשים Azure CLI מחובר, תפקיד `Cognitive Services Speech User` על משאב בעל custom domain, ו־FFmpeg. אין צורך במפתח או בסוד:

```bash
az login
export AZURE_SPEECH_RESOURCE_ID="$(az cognitiveservices account show \
  --name <resource-name> \
  --resource-group <resource-group> \
  --query id -o tsv)"
export AZURE_SPEECH_REGION="<region>"
npm run generate:speech
```

הקולות הם `he-IL-HilaNeural`, `en-US-AnaNeural` ו־`en-GB-MaisieNeural`. שני הקולות באנגלית מסווגים על ידי Azure כקולות ילדה, וברירת המחדל היא `en-US-AnaNeural`. ל־Azure אין קול ילד עברי מסווג במערך הזמין, ולכן העברית נשארת עם `he-IL-HilaNeural`, קול נשי מאושר שאינו מסווג כקול ילד. שמות הקולות נשמרים גם במפת המקטעים ונבדקים על ידי אימות נכסי הדיבור, כדי שהמחולל והקבצים המקומיים לא יסטו ממדיניות הקול. המחולל מקבל אסימון Entra דרך `AzureCliCredential`, מייצר שמע חד־ערוצי בקצב 24kHz, ואינו מדפיס או שומר את האסימון. אין באפליקציה ממשק הקלטה, והקבצים אינם נאספים מהמשתמש.

לאחר יצירה מלאה, אפשר לתקן הגייה עברית נקודתית שאומתה בלי לשלם שוב על כל הקטלוג. מגדירים `AZURE_SPEECH_REGENERATE_MATCH` למילת המקור הלא־מנוקדת ומריצים שוב את אותה פקודה. המחולל יוצר מחדש רק את המקטעים העבריים התואמים, מחלץ את שאר המקטעים מן הספרייט הקיים, ובונה מחדש את הספרייט והמפה לפני הרצת `npm run validate:speech`.

## הערה חשובה

הפרויקט מיועד לחוויה משחקית ולמידה עדינה בלבד. הוא לא מציג טענות רפואיות, טיפוליות או אבחוניות, אינו טוען שהוא ממצה או מאושר על ידי אף גורם מקצועי, ואינו תחליף לשיחה ולמשחק משותף עם מבוגר.
