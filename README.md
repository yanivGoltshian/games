# שלום שון

אפליקציית PWA מקומית, חמה וידידותית למגע עבור שון, ילד בן 2.5 שעדיין לא קורא. הפרויקט בנוי ב־Vite + React + TypeScript, מותאם קודם כול לאייפד ולטלפון, וכולל חמישה משחקים פעילים עם התקדמות אדפטיבית מקומית בלבד.

## פקודות

```bash
npm install
npm run dev
npm run test
npm run lint
npm run typecheck
npm run build
npm run preview
```

## התקנה על אייפד (הערה למבוגרים בלבד)

### Safari
1. פותחים את האפליקציה.
2. לוחצים על כפתור השיתוף.
3. בוחרים **הוספה למסך הבית**.
4. מאשרים את השם “שלום שון”.

### Chrome באייפד
1. פותחים את התפריט או Share.
2. בוחרים **Add to Home Screen**.
3. מאשרים את ההתקנה.

אחרי טעינה ראשונה, ה־service worker שומר קבצים מקומיים כך שאפשר לשחק גם בלי חיבור רשת. כל ההערות האלה על התקנה, אופליין וקול מופיעות רק במסך המבוגרים באפליקציה עצמה (אזור למבוגרים), ולעולם לא במסך הילד.

## מה יש בפנים

- **בית לילד**: קרוסלת פורטלים אופקית עם CSS scroll-snap, איור ענק אחד לכל משחק עם זהות ויזואלית יציבה (צבע, קומפוזיציה, דמות). פורטל אחד גלוי בטלפון, כ־1–2 באייפד. בלי טקסט הוראות, בלי כפתור התקנה, בלי הערת פרטיות ובלי לוח מחוונים של התקדמות – כל אלה עברו למסך המבוגרים בלבד.
- **5 משחקים פעילים**, עם רמה 1 מוגבלת בכוונה לשתי ברירות בלבד (שומעים/סופרים), שני פריטים ושני סלים (מיון), שתי חתיכות (פאזל) ושני זוגות (זיכרון):
  - שומעים ובוחרים
  - כמה יש?
  - ממיינים צבעים וצורות
  - פאזל קטן עם snap סלחני
  - זוגות זיכרון
- **מסך משחק לילד**: משימה אחת ברורה בכל זמן נתון, ובראש המסך רק שני כפתורים גדולים – חזרה לבית והשמעה חוזרת. שער המבוגרים אינו מופיע כלל בתוך משחק.
- **ספריית איורים מקורית** תחת `src/art`: כל האובייקטים, הצורות, חמשת הפורטלים, סצנות הפאזל, הכלבלב הקטלוגי (מסקוט), הקונפטי והבקרות הווקטוריות מצוירים ב־SVG מקומי, בלי אימוג׳י ובלי משאבים חיצוניים, עם מזהי איור יציבים ותוויות נגישות.
- **חגיגת הצלחה עדינה**: כל תשובה נכונה מציגה חיזוק לא־חוסם למשך כ־1.6–2 שניות עם המסקוט, קונפטי, צליל וריטוט אופציונלי, ומתקדם אוטומטית (או בלחיצה בכל מקום). לאחר רצף הצלחות קטן מופיעה חגיגה מעט עשירה יותר עם רגע חזותי שקט שמזמין משחק אמיתי יחד – בלי לחץ ובלי מונים.
- **שבחים מרכזיים ודטרמיניסטיים** ב־`src/content/praise.ts`: מזהים יציבים, בחירה טהורה ונבדקת (`praise.test.ts`), עברית ואנגלית תואמות לשפת הלמידה הפעילה, ורצף דיבור שתמיד משמיע קודם את שם היעד/התשובה ורק אז את השבח.
- **התקדמות אדפטיבית** ב־localStorage: ניסיונות, הצלחות, רמת שליטה, עלייה עדינה ברמה וחזרה על מושגים חלשים – מוצגת רק במסך המבוגרים.
- **אזור למבוגרים** שנפתח בלחיצה ארוכה של כ־2.7 שניות על גלגל השיניים, ורק שם.
- קול דרך SpeechSynthesis, צלילי חיזוק דרך Web Audio, ריטוט קליל אופציונלי, מצב שקט, פחות תנועה (גם לפי הגדרת מבוגר וגם לפי `prefers-reduced-motion`), והגדרות שפה.
- PWA מקומי בלבד, בלי אנליטיקות, בלי חשבונות ובלי קישורים חיצוניים.

## ארכיטקטורה קצרה

- `src/art` – ספריית קומפוננטות SVG מקורית: אובייקטים וחיות, צורות/צבעים, איורי הפורטלים, סצנות פאזל, מסקוט, קונפטי ובקרות. מזהי `data-art-id` יציבים לכל איור.
- `src/content` – מטא־דאטה מרכזי למושגים, משחקים, סצנות פאזל, ובנק השבחים הדטרמיניסטי.
- `src/domain` – RNG דטרמיניסטי, לוגיקת סיבובים (כולל אילוצי רמה 1), התקדמות, ולידציה/מיגרציה של persistence.
- `src/services` – שמירה מקומית, הקראה (כולל רצף "יעד ואז שבח" בטוח לביטול), וצלילי חיזוק/ריטוט.
- `src/components` – מעטפת UI לילד (בית, מסגרת משחק, חגיגת הצלחה), שער למבוגרים, פאנל מבוגרים, עזרי גרירה משותפים.
- `src/games` – כל אחד מחמשת המשחקים.
- `public/sw.js` ו־`public/manifest.webmanifest` – יכולות PWA ואופליין.

## פרטיות

כל המידע נשמר רק על המכשיר המקומי. אין שרת, אין שליחת נתונים, אין איסוף אנליטיקות, אין חשבונות ואין אחסון חיצוני.

## גישה מבוססת־ראיות, לא קלינית

העיצוב הפדגוגי של האפליקציה שואב השראה ממקורות חינוכיים ומקצועיים ציבוריים על התפתחות שפה, דו־לשוניות ומסכים אצל פעוטות. הרשימה הבאה **אינה ממצה, אינה מהווה אישור, הסמכה או המלצה קלינית** מצד הגורמים המוזכרים, ואינה תחליף לייעוץ של איש/אשת מקצוע (למשל קלינאי/ת תקשורת, רופא/ת ילדים או גורם חינוכי מוסמך). האפליקציה אינה מציגה ואינה מתיימרת להציג אבחון, טיפול או ניטור התפתחותי.

מקורות שהעקרונות התומכים בעיצוב (למידה קליטתית קודם, תיוג במגע, מודלינג קצר, חזרה עם גיוון, קיבוץ סמנטי, הנחיה בצעד אחד, השהיה לפני רמז/מודלינג, חיזוק חיובי מיידי, תורנות, וחשיפה דו־לשונית) מתכתבים איתם:

- ASHA – אבני דרך בתקשורת לגילאי 2–3: <https://www.asha.org/public/developmental-milestones/communication-milestones-2-to-3-years/>
- האקדמיה האמריקאית לרפואת ילדים (HealthyChildren.org) – התפתחות שפה בגיל שנתיים: <https://www.healthychildren.org/English/ages-stages/toddler/Pages/Language-Development-2-Year-Olds.aspx>
- האקדמיה האמריקאית לרפואת ילדים – חמשת ה־C's לשימוש נבון במסכים: <https://www.healthychildren.org/English/family-life/Media/Pages/kids-and-screen-time-5-cs-questions-for-toddlers-and-preschoolers.aspx>
- The Hanen Centre – טיפים לתמיכה בלומדי שתי שפות: <https://www.hanen.org/information-tips/tips-for-supporting-dual-language-learners>
- The Hanen Centre – ארבעת ה־S's להבלטת מילים חדשות: <https://www.hanen.org/information-tips/make-words-stand-out-with-the-four-ss>
- ZERO TO THREE – "It Takes Two: The Roots of Language Learning": <https://www.zerotothree.org/resource/it-takes-two-the-roots-of-language-learning/>
- WHO / Nurturing Care Framework for Early Childhood Development: <https://www.who.int/publications/i/item/9789241514064>

### שיחה ומשחק משותף בעולם האמיתי

שום אפליקציה, כולל זו, אינה תחליף לשיחה, הקראה, ומשחק משותף עם מבוגר. הסיבובים במשחקים קצרים בכוונה, ואחרי רצף הצלחות קטן מופיעה חגיגה עם רגע חזותי רגוע שמזמין לצאת לשחק יחד. מומלץ שההורה/המטפל יהיה נוכח, ישתתף וימשיך את החוויה בעולם האמיתי – שיחה, הצבעה על עצמים אמיתיים, חזרה על מילים וקריאה משותפת תורמות הרבה יותר מזמן מסך עצמאי.

## מגבלת קול נוכחית (V1) ותפר השירות לעתיד

כרגע ההקראה משתמשת ב־SpeechSynthesis של הדפדפן (Safari/Chrome), ולכן:
- איכות הקול תלויה בקולות המותקנים במכשיר.
- ייתכן שבמכשירים שונים יהיו הבדלים בהגייה ובזמינות הקולות.
- אם אין קול זמין, המשחקים עדיין עובדים במלואם עם תמונות ומגע – שום דבר לא תלוי בקול בלבד.

הארכיטקטורה בנויה סביב **מזהי תוכן ואודיו יציבים** (שדה `audio` ב־`LearningConcept`, מזהי `praise-he-01` וכדומה בבנק השבחים) כדי לאפשר בעתיד מעבר לקבצי הקלטה אמיתיים בלי לשנות אף קורא (game/UI):

1. מוסיפים קבצים מקומיים לתיקייה קבועה, למשל `public/audio/he/` ו־`public/audio/en/`, בשם התואם למזהה היציב (למשל `ball.mp3`, `praise-he-01.mp3`).
2. ממלאים לכל מושג ב־`src/content/concepts.ts` את שדה `audio` עם הנתיב המתאים, ובאותו האופן אפשר להרחיב את `src/content/praise.ts`.
3. משנים ב־`src/services/speech.ts` את השירות כך שיבדוק קודם אם קיים קובץ מוקלט למזהה הנוכחי, וישמיע אותו (דרך `<audio>`) לפני נפילה חזרה ל־SpeechSynthesis.
4. אפשר להשאיר את SpeechSynthesis כ־fallback קבוע למקרה שחסר קובץ – כך שאין "הכול או כלום".

**חשוב**: זהו רק תפר טכני (service seam) שמאפשר להחליף ספק קול בעתיד. הוא **אינו** דורש הקלטות שנבדקו על ידי קלינאי/ת תקשורת, ואינו כולל וממילא **לא נדרש** ממשק הקלטה בתוך האפליקציה, IndexedDB או MediaRecorder – קבצי השמע (אם וכאשר יתווספו) הם נכסים סטטיים שנוספים מראש לתיקיית `public`, לא הקלטות שנאספות מהמשתמש/ת.

## הערה חשובה

הפרויקט מיועד לחוויה משחקית ולמידה עדינה בלבד. הוא לא מציג טענות רפואיות, טיפוליות או אבחוניות, אינו טוען שהוא ממצה או מאושר על ידי אף גורם מקצועי, ואינו תחליף לשיחה ולמשחק משותף עם מבוגר.
